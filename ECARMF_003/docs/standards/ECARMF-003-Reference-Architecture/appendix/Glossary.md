# ECARMF-003 Glossary

**Bounded Context:** A domain boundary that owns its business rules, data model, APIs, events, and lifecycle.

**Canonical API:** A standardized API conforming to ECARMF URI, request, response, error, security, telemetry, and testing rules.

**Canonical Service:** A service implementing one business responsibility and conforming to ECARMF service architecture.

**Domain Event:** An immutable fact representing a completed business action.

**Requirement Traceability:** The ability to link code, tests, APIs, database objects, and documentation back to ECARMF requirement IDs.
