# Contributing to ECARMF

Contributions are welcome.

## Contribution Areas

- Documentation
- Mathematical models
- JSON schemas
- Python implementation
- Sample datasets
- Tests
- Diagrams
- Use-case examples

## Contribution Process

1. Open an issue describing the proposed change.
2. Create a branch.
3. Submit a pull request.
4. Include documentation and tests where applicable.
5. Wait for review before merge.
