# Chapter 6 — Canonical Entity Catalog

## Requirement CAT-0001
Every implementation shall use only Canonical Entities defined within ECARMF or approved extension standards.

## Entity Domains

| Domain | Code | Examples |
|---|---|---|
| Business | BUS | EDA, EDC, Portfolio, Organization, Stakeholder, Project |
| Financial | FIN | Capital, Equity, Debt, Revenue, Expense, Cash Flow, Token |
| Economic | ECO | Economic Context, Market, Inflation, Interest Rate, Regulation |
| Risk | RSK | Risk, Opportunity, Exposure, Control, Mitigation |
| Operational | OPR | Process, Activity, Task, Resource, Equipment, Event |
| Governance | GOV | Standard, Requirement, Policy, Compliance, Audit, Evidence |
| Analytical | ANA | Threshold, Decision, Scenario, Forecast, Simulation, Score |
| Technical | TEC | Knowledge Graph, Digital Twin, Dataset, API, Service, Schema |

## Requirement CAT-0100
Every Canonical Entity shall define identifier, definition, domain, parent, child rules, attributes, lifecycle, relationships, requirements, and version history.
