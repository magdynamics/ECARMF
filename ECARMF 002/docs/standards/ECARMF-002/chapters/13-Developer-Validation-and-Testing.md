# Chapter 13 — Developer Validation & Testing

## Requirement TST-0001
Every normative requirement shall have one or more corresponding verification tests.

## Testing Hierarchy
Requirement Test → Unit Test → Component Test → Integration Test → Knowledge Graph Test → Digital Twin Test → API Test → Performance Test → Security Test → Conformance Test.

## Requirement UNT-0001
Minimum unit test coverage shall be 95% of business logic.

## Requirement TRT-0001
Every test case shall reference at least one ECARMF requirement.

## Requirement CERT-0001
Certification suite mandatory for ECARMF certification.
