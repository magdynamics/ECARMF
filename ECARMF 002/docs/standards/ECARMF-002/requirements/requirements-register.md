# ECARMF-002 Requirement Register

| ID | Domain | Requirement | Priority |
|---|---|---|---|
| UEM-0001 | Universal Entity Model | Every ECARMF managed object shall inherit from the Universal Base Entity | Mandatory |
| KG-0001 | Knowledge Graph | Every ECARMF implementation shall maintain a logical knowledge graph | Mandatory |
| EDC-0001 | Economic Digital Cell | Every ECARMF Entity shall be decomposable into one or more Economic Digital Cells where applicable | Mandatory |
| UNI-0001 | Universal Laws | Every EDC shall possess globally unique immutable identity | Mandatory |
| CAT-0001 | Entity Catalog | Every implementation shall use only Canonical Entities or approved extensions | Mandatory |
| UAL-0001 | Attributes | Every Canonical Entity shall inherit the Universal Attribute Library | Mandatory |
| CRL-0001 | Relationships | Every relationship shall be instantiated as a Relationship Entity | Mandatory |
| ULS-0001 | Lifecycle | Every Canonical Entity shall implement the Universal Lifecycle | Mandatory |
| MMC-0001 | Constraints | Every implementation shall enforce ECARMF constraints | Mandatory |
| EIB-0001 | Implementation | Every implementation shall conform to the Engineering Implementation Blueprint | Mandatory |
| TST-0001 | Testing | Every normative requirement shall have verification tests | Mandatory |
| AIE-0001 | AI | AI-assisted implementation shall preserve ECARMF semantic integrity | Mandatory |
| CON-0001 | Conformance | Only implementations satisfying mandatory requirements may claim ECARMF compliance | Mandatory |
